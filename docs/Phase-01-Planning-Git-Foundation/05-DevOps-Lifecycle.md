# 1. Objective

The purpose of this document is to explain the organizational structure involved in delivering software using DevOps practices.

In a real enterprise, software delivery is not the responsibility of a single person or team. Multiple teams collaborate throughout the Software Development Life Cycle (SDLC) and DevOps Lifecycle.

This document explains:

- Organization hierarchy
- Team responsibilities
- Ownership of each activity
- Communication flow
- Release approval process
- Real production workflow
